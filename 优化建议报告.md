## 三、各模块优缺点分析

### 🔴 高优先级 — 需要完善

#### 1. 语音导览（heritageDetail / scenicDetail）
**优点：**
- 已接入百度TTS云函数，按需合成语音
- 支持本地 audioUrl 优先播放，避免重复合成
- 页面卸载时自动停止播放，节省资源

**缺点/风险：**
- 百度TTS云函数需要配置百度API Key，需额外申请
- 方言（四川话）TTS质量有限，建议录制真实音频上传到云存储
- 合成语音每次调用消耗百度配额，高并发时成本上升
- 文档要求"四川话/普通话双语切换"，当前仅实现了单语音

**优化建议：**
```javascript
// 建议数据结构：将 single audioUrl 改为双语音
audioUrl: 'https://...',           // 普通话
audioSichuanUrl: 'https://...',    // 四川话（可选）
audioMode: 'sichuan' | 'mandarin'  // 当前播放模式
```
同时在云存储中预上传 `audioUrl` 和 `audioSichuanUrl` 两个字段的音频文件。

#### 3. 打卡功能（checkin.vue / myCheckin.vue）
**优点：**
- 完整实现了定位验证 → 拍照上传 → 打卡提交 → 海报生成的闭环
- 支持小程序码海报分享
- 打卡记录有时间轴展示，UI体验良好

**缺点/风险：**
- 定位验证依赖云函数 `checkLocation` 计算距离，需确保云数据库中非遗/景点有准确坐标
- 500米范围在GPS信号弱时可能误判
- 打卡图片上传到云存储后，海报生成依赖 Canvas 绘制，真机表现可能与模拟器有差异
- 没有防重复打卡逻辑（同一天同一地点多次打卡）

**优化建议：**
```javascript
// 防重复打卡：查询今天是否已有相同地点打卡记录
const today = new Date().toDateString();
const existing = await db.collection('checkin')
  .where({ openId, targetId: this.id, type: this.type })
  .where('createTime >= ' + new Date(today).getTime())
  .count();
if (existing.total > 0) {
  return uni.showToast({ title: '今日已打卡，请明天再来', icon: 'none' });
}
```

## 四、架构优化建议

### 4.1 统一错误处理
建议在 `App.vue` 中全局处理云函数错误：

```javascript
// App.vue
Vue.config.errorHandler = (err, vm, info) => {
  console.error('Global Error:', err, info);
  if (err.message && err.message.includes('cloud')) {
    uni.showToast({ title: '网络异常，请稍后重试', icon: 'none' });
  }
};
```

### 4.2 登录态与 openId 管理
当前代码中 `openId` 均在需要时调用 `wx.cloud.getTempFileURL` 获取，建议：

```javascript
// 在 App.vue onLaunch 中预先获取并缓存
onLaunch() {
  wx.cloud.init({ env: 'cloud1-xxx' });
  wx.cloud.callFunction({ name: 'getOpenId' }).then(res => {
    uni.setStorageSync('openId', res.result.openid);
  });
}
```

### 4.3 图片加载优化
小程序中大量图片会导致性能问题，建议：
- 统一使用云存储的图片（已使用 `cloud://` 路径会自动替换为临时链接）
- 在列表页启用图片懒加载
- 对封面图使用合适的 `mode="aspectFill"` 避免拉伸

### 4.4 云函数并发限制
微信云开发免费版云函数月调用上限2万次，建议：
- 非实时性数据（如热门推荐）添加本地缓存（`storage.js` 中已有 `setWithExpire`）
- 批量查询使用 `Promise.all` 而非串行调用（如搜索页面同时查非遗和景点）

### 4.5 代码规范统一
当前各页面风格基本统一，但有以下不一致：
- 部分页面用 `<uni-icons>`，部分用 emoji 或纯文字
- 部分使用 `async/await`，部分使用 `.then()`
- 建议统一改为 `async/await` + `try/catch`
