# 小蜀记 - 四川非遗文旅数字导览小程序

## 概述

**小蜀记**是一款基于uni-app与微信云开发的四川非物质文化遗产文旅数字导览小程序，为用户提供非遗项目展示、景点导览、打卡记录、路线规划等一站式文旅服务，助力非遗文化的数字化传播与活态传承。

---

## 一、功能架构

```
├── 首页（首页轮播 + 非遗推荐 + 景点推荐）
├── 探索页（功能入口集合）
├── 路线规划（多模式路线规划）
├── 我的（个人中心 + 收藏 + 打卡 + 浏览历史 + 反馈）
└── TabBar
    ├── 首页
    ├── 非遗列表
    ├── 景点列表
    └── 我的
```

## 二、核心功能详述

### 2.1 首页模块

**路径**：`pages/index/index.vue`

| 功能 | 说明 |
|------|------|
| 轮播图 | 展示精选非遗/景点内容，支持自动轮播 |
| 非遗推荐 | 展示6条热门非遗项目卡片，点击进入详情 |
| 景点推荐 | 展示6条热门景点卡片，点击进入详情 |
| 快速入口 | 导航至打卡页、路线规划页、探索页 |

**交互流程**：
1. 页面加载时自动请求推荐数据
2. 点击卡片跳转至对应详情页
3. 支持下拉刷新更新数据

### 2.2 非遗模块

#### 2.2.1 非遗列表页

**路径**：`pages/heritage/heritageList.vue`

| 功能 | 说明 |
|------|------|
| 分类筛选 | 按类别筛选非遗项目（传统技艺、民俗、传统医药等） |
| 瀑布流展示 | 双列瀑布流布局，封面图自适应高度 |
| 浏览量显示 | 每条卡片显示浏览量 |
| 点击跳转 | 点击卡片进入非遗详情页 |

#### 2.2.2 非遗详情页

**路径**：`pages/heritage/heritageDetail.vue`

| 功能 | 说明 |
|------|------|
| 封面展示 | 大图封面 + 标题 + 分类标签 |
| 详情内容 | 展示非遗项目的历史、特点、传承价值等 |
| 语音导览 | 调用百度TTS将文字转为语音，支持播放/暂停 |
| 地图关联 | 显示关联景点位置，支持一键导航 |
| 收藏功能 | 点击收藏/取消收藏，记录到数据库 |
| 浏览历史 | 自动记录浏览历史到数据库 |

**语音导览流程**：
```
用户点击"语音导览" → 调用tts-guide云函数 → 获取百度Access Token
→ 调用百度TTS API合成语音 → 上传到微信云存储
→ 返回HTTPS临时链接 → 小程序Audio组件播放
```

### 2.4 打卡模块

#### 2.4.1 打卡页

**路径**：`pages/checkin/checkin.vue`

| 功能 | 说明 |
|------|------|
| 位置验证 | 调用checkLocation云函数验证用户是否在有效范围内 |
| 拍照上传 | 支持拍照或从相册选择打卡照片 |
| 备注信息 | 支持添加简短备注（选填） |
| 打卡记录 | 打卡成功后写入checkin集合 |

**打卡条件**：
- 用户位置与目标地点距离需在500米以内
- 需授权位置权限
- 需上传至少一张照片

**数据存储**：
```javascript
// checkin 集合字段
{
  _id: string,          // 文档ID
  _openid: string,      // 用户openId
  type: 'heritage' | 'scenic',  // 打卡类型
  targetId: string,     // 关联的非遗/景点ID
  targetName: string,   // 关联名称
  photo: string,        // 云存储文件ID
  remark: string,       // 备注
  location: object,     // 打卡位置 { latitude, longitude }
  createTime: number    // 创建时间戳
}
```

## 三、技术实现

### 3.1 技术栈

| 类别 | 技术 |
|------|------|
| 框架 | uni-app + Vue2 |
| 云开发 | 微信云开发（云数据库、云函数、云存储） |
| 地图 | 腾讯地图SDK |
| 语音 | 百度TTS文字转语音 |
| UI | uni-ui组件库 |

### 3.2 云函数

| 云函数 | 功能 |
|--------|------|
| `tts-guide` | 百度TTS文字转语音，上传云存储 |
| `checkLocation` | 验证用户位置是否在有效打卡范围内 |
| `addCheckin` | 提交打卡记录 |
| `getMyCheckin` | 获取用户打卡记录列表 |
| `favorite-add` | 添加收藏 |
| `favorite-remove` | 取消收藏 |
| `favorite-list` | 获取收藏列表（支持筛选） |
| `geoReverse` | 逆地址解析（坐标转地址） |
| `initViewCount` | 初始化浏览量字段 |

### 3.3 云数据库集合

| 集合 | 用途 |
|------|------|
| `heritage` | 非遗项目数据 |
| `scenic` | 景点数据 |
| `checkin` | 用户打卡记录 |
| `user_favorites` | 用户收藏记录 |
| `browse_history` | 用户浏览历史 |
| `feedback` | 用户反馈 |
| `tts_cache` | TTS Token缓存 |

### 3.4 云存储路径

| 路径 | 用途 |
|------|------|
| `images/heritage/` | 非遗封面图 |
| `images/scenic/` | 景点封面图 |
| `images/checkin/` | 打卡照片 |
| `images/audio/` | TTS合成的语音文件 |
| `images/banner/` | 轮播图 |

## 四、用户体验设计

### 4.1 隐私保护

- **地图默认定位**：页面加载时默认显示成都中心点，不自动申请位置权限
- **用户主动触发**：仅在用户主动点击"我的位置"按钮时才申请定位权限

### 4.2 交互反馈

- **加载状态**：请求数据时显示loading动画
- **空状态**：数据为空时显示友好提示和引导按钮
- **错误处理**：操作失败时显示错误原因和重试提示

### 4.3 视觉设计

- **主题色**：川红 #D93025，搭配竹韵青 #1A7F64
- **卡片风格**：圆角白色卡片 + 轻微阴影
- **时间轴**：红色（非遗）/ 蓝色（景点）圆点标识
