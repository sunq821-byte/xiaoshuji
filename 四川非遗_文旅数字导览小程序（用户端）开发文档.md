# 四川非遗/文旅数字导览小程序（用户端）开发文档

本项目是用uni-app（Vue2）框架开发的、聚焦四川非遗与文旅资源的微信小程序，后端采用微信云开发（含云数据库、云函数、云存储），集成百度语音合成API及腾讯地图SDK，实现非遗导览、景点介绍、方言语音导览、文旅打卡及路线规划等功能，助力四川非遗文化数字化传承与智慧文旅体验升级。

> **版本说明**：原规划的AR识别功能已移除（依赖微信AR插件，环境复杂），改为以语音导览、地图导航、打卡功能为核心体验。

# 一、开发基础信息

## 1.1 技术栈

前端：uni-app（Vue2）+ 微信小程序原生能力
后端：微信云开发（云数据库、云函数、云存储）
依赖插件/API：百度语音合成API（TTS）、腾讯地图SDK
UI组件：uni-ui（uni-icons 等）
工具封装：utils/cloudCall.js（云函数调用 + 云存储图片处理）、utils/storage.js（本地存储）、utils/tencentMapService.js（地图服务）

## 1.2 开发环境

开发工具：HBuilderX（安装uni-app插件、Vue语法提示）、微信开发者工具
测试环境：微信小程序模拟器、真机测试（至少1台安卓/苹果手机）
云环境：微信云开发免费版（数据库5GB、存储5GB、云函数月调用2万次）
云环境ID：cloud1-7gabd815fd2c236e.636c-cloud1-7gabd815fd2c236e-1418729882

> ⚠️ **注意**：uni-app + HBuilderX 项目慎用第三方 npm 组件库（如 TDesign），依赖链解析易出错。UI 组件优先使用 uni-ui（已内置）或原生小程序组件。

## 1.3 核心开发原则

1. 优先实现核心功能，不做冗余交互；
2. 页面风格统一（简约、贴合文旅非遗调性，以四川特色色调为主）；
3. 兼容微信小程序（优先）和Web端，无需额外适配其他平台；
4. 接口调用统一封装，便于后期联调和维护；
5. 代码注释清晰，便于团队协作（若有队友配合）。

## 模块5：路线规划与地图导航（pages/route/route.vue）

> **结构调整**：原规划的 routeList + routeDetail 双页面已合并为单一 `route.vue`，同时承担**路线地图展示**和**景点导航**两种场景。

### 3.5.1 页面功能

1. **地图主界面**：展示腾讯地图，默认中心定位成都（30.67, 104.06），不自动申请位置权限；
2. **景点搜索**：顶部搜索框，搜索四川省内景点/地点（调用腾讯地图 getSuggestion API）；
3. **起点/终点设置**：支持手动选择起点和终点，或由导航模式自动填充；
4. **路线规划**：调用腾讯地图 direction API，规划驾车路线，在地图上绘制路线线条（polyline）；
5. **地图标记**：起点（蓝色）、终点（红色）在地图上显示标记；
6. **我的位置**：用户主动点击按钮后才申请定位权限，获取并展示当前位置；
7. **导航模式**（从景点详情页进入）：自动将景点设为终点，自动定位用户位置为起点，直接规划路线。

### 3.5.2 两种入口场景

| 场景 | 入口 | 参数 | 行为 |
|------|------|------|------|
| 普通路线规划 | 首页导航/探索页 | 无 | 展示地图，用户手动搜索选点 |
| 景点导航 | 景点详情页"地图导航"按钮 | `type=navigate&name=...&lat=...&lng=...` | 自动设置终点，自动获取用户位置，规划路线 |

### 3.5.3 接口调用

1. 地点搜索（前端直接调用腾讯地图 JS SDK）

## 4.2 云存储图片处理（utils/cloudCall.js）

> **重要**：云存储返回的 `cloud://` 路径无法直接在小程序 `<image>` 组件中显示，必须先转换为临时链接。

```javascript
// 获取单个或多个云存储文件的临时链接
export const getTempFileURL = async (fileID) => {
  if (!fileID) return '';
  const ids = Array.isArray(fileID) ? fileID : [fileID];
  const res = await wx.cloud.getTempFileURL({ fileList: ids });
  return Array.isArray(fileID)
    ? res.fileList.map(f => f.tempFileURL)
    : res.fileList[0].tempFileURL;
};

// 批量处理数据中的图片字段（列表或单条数据均可）
export const processImageFields = async (data, imageFields = []) => {
  if (!data) return data;
  const list = Array.isArray(data) ? data : [data];
  const processed = await Promise.all(list.map(async (item) => {
    const newItem = { ...item };
    for (const field of imageFields) {
      if (newItem[field]) {
        if (Array.isArray(newItem[field])) {
          newItem[field] = await getTempFileURL(newItem[field]);
        } else {
          newItem[field] = await getTempFileURL(newItem[field]);
        }
      }
    }
    return newItem;
  }));
  return Array.isArray(data) ? processed : processed[0];
};
```

**使用示例**：
```javascript
import { cloudCall, processImageFields } from '@/utils/cloudCall.js';

// 处理列表数据中的 coverImage 字段
const res = await cloudCall('getHeritageList', { page: 1 });
this.list = await processImageFields(res.list, ['coverImage']);

// 处理详情页多个图片字段
const detail = await cloudCall('getScenicDetail', { id });
this.scenic = await processImageFields(detail, ['coverImage', 'gallery']);
```
