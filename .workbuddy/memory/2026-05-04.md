## 浏览次数界面不更新问题修复

### 问题现象
- 进入非遗/景点详情页时，控制台显示"浏览量+1"
- 但界面始终显示 `1 次`，不会递增（0→1→2→3...）

### 根本原因
**Vue 2 响应式限制**：
1. 数据库文档中没有 `viewCount` 字段时，`res.data` 中也不存在 `viewCount` 键
2. `processImageFields` 返回浅拷贝 `{ ...data }`，同样没有 `viewCount` 属性
3. `this.heritageDetail = result` 赋值时，Vue 2 的 `Object.defineProperty` 只会对**已存在**的属性建立响应式追踪
4. 后续 `this.heritageDetail.viewCount = 1` 相当于给对象添加新属性，Vue 2 检测不到变化，视图不更新

### 修复方案
在 `loadHeritageDetail` / `loadScenicDetail` 中，给 `viewCount` 设置默认值 `0`，确保 Vue 初始化时就能追踪：

```javascript
this.heritageDetail = await processImageFields(res.data, ['coverImage', 'image', 'gallery']);
// 确保 viewCount 存在，Vue2 才能建立响应式追踪
if (this.heritageDetail.viewCount === undefined) {
    this.heritageDetail.viewCount = 0;
}
```

### 修改文件
- `pages/heritage/heritageDetail.vue`（loadHeritageDetail 方法）
- `pages/scenic/scenicDetail.vue`（loadScenicDetail 方法）

## 浏览次数问题最终修复（方案A）

### 根本问题（比之前分析更深入）

**问题1：云数据库 `db.command.inc(1)` 对不存在的字段不生效**
- `update({ data: { viewCount: _.inc(1) } })` 当文档中没有 `viewCount` 字段时，**静默忽略，不创建字段，不报错**
- `update()` 成功不抛异常 → 不进入 catch → `console.log('浏览量+1')` 正常打印
- 但数据库里 `viewCount` 从未被创建 → 下次进入页面仍然没有这个字段

**问题2：Vue 2 响应式丢失**
- `processImageFields` 返回 `{ ...data }` 浅拷贝，如果数据库没有 `viewCount`，拷贝后也没有这个键
- Vue 2 的 `Object.defineProperty` 只对初始化时已存在的属性建立响应式追踪
- 后续动态添加的属性不会被追踪，视图不更新

### 修复措施

**1. `loadHeritageDetail` / `loadScenicDetail` 修改：**
```javascript
// 先存到临时变量，确保 viewCount 存在后再赋值给 this.heritageDetail
let detail = await processImageFields(res.data, ['coverImage', 'image', 'gallery']);
if (detail.viewCount === undefined) {
    detail.viewCount = 0;
}
this.heritageDetail = detail;  // Vue 初始化时能追踪到 viewCount
```

**2. `incrementViewCount` 修改：**
```javascript
// 不用 _.inc()，直接计算新值并 set
const newCount = (this.heritageDetail?.viewCount || 0) + 1;
await db.collection('heritage').doc(this.heritageId).update({
    data: { viewCount: newCount }
});
// 用 $set 确保 Vue2 响应式触发视图更新
this.$set(this.heritageDetail, 'viewCount', newCount);
```

### 修改文件
- `pages/heritage/heritageDetail.vue`（loadHeritageDetail + incrementViewCount）
- `pages/scenic/scenicDetail.vue`（loadScenicDetail + incrementViewCount）

### 验证逻辑
1. 首次进入：数据库无 viewCount → 本地默认 0 → increment 计算 0+1=1 → set 到数据库 → $set 更新视图 → 界面显示 1
2. 再次进入：数据库 viewCount=1 → 本地读取 1 → increment 计算 1+1=2 → set 到数据库 → $set 更新视图 → 界面显示 2
3. 第三次进入：数据库 viewCount=2 → 本地读取 2 → +1=3 → 界面显示 3

## 浏览量实现方案优化（改用 _.inc(1) 原子递增）

### 问题
之前用前端计算新值（`current + 1`）再写入的方式存在并发问题：多用户同时进入时可能读到相同值，导致少算。

### 修复方案
改用 `db.command.inc(1)` 原子递增：
- 数据库层面原子 +1，无并发问题
- 字段不存在时自动创建并设为 1
- 递增后重新读取最新值，通过 `this.$set` 触发界面更新

### 修改文件
- `pages/heritage/heritageDetail.vue`：`incrementViewCount` 方法
- `pages/scenic/scenicDetail.vue`：`incrementViewCount` 方法
- 同时清理了两个 `loadDetail` 方法中的 `browseCount` 初始化逻辑（不再需要）

### 当前实现逻辑
1. 进入详情页 → `loadHeritageDetail` / `loadScenicDetail`
2. 读取数据（含 `browseCount`，若没有则为 `undefined`）
3. 赋值给 `this.heritageDetail` / `this.scenicDetail`（Vue 响应式）
4. 调用 `incrementViewCount()`
5. `incrementViewCount` 中用 `_.inc(1)` 原子递增数据库字段
6. 重新读取最新值，用 `this.$set` 更新界面

### 注意事项
- 需要确保数据库集合有写权限（前端可读写）
- 如果权限不足，需要改用云函数实现递增
