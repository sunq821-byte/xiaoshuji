# 2026-05-05 日志

## 语音导览功能优化 (2026-05-05)

### 需求
语音导览失败时提示用户重试，而不是显示"导览功能不可用"

### 修改文件
1. **pages/heritage/heritageDetail.vue** - 非遗详情页
2. **pages/scenic/scenicDetail.vue** - 景点详情页

### 实现方案
1. 添加 `audioRetryCount` 计数器（最大重试2次）
2. 新增 `handleAudioError(message)` 方法：
   - 显示 `uni.showModal` 弹窗，询问"是否重试"
   - 用户选择"重试"则重新调用 `startAudioGuide()`
   - 用户选择"取消"则重置计数
   - 超过重试次数后显示 Toast 提示"请稍后重试"
3. 修改 `startAudioGuide()` 失败时调用 `handleAudioError`
4. 修改 `playAudioFromUrl()` 音频错误时调用 `handleAudioError`（区分网络错误和其他错误）

### 用户体验改进
- 失败后弹出对话框："语音生成失败，是否重试？"
- 提供明确的"重试"和"取消"选项
- 最多可重试2次，避免无限循环
- 超过次数后提示用户稍后再试
