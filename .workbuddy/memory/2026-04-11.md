## 腾讯地图路线规划 polyline 解码修复（17:32）

### 问题现象
- 地图上未绘制导航路线
- 预计时间显示为 0 分钟
- 渲染层错误：`LatLng.lat 取值应在 [-90, 90] 区间内`

### 根本原因
腾讯地图返回的 `polyline` 数组格式：`[lat1, lng1, dlat2, dlng2, dlat3, dlng3, ...]`

| 组成部分 | 含义 | 示例 |
|---------|------|------|
| 第一对 | 真实坐标（度数） | 31.006261, 103.614781 |
| 后续值 | 差分值（单位 1e-6 度） | 169, 433, 53, 120... |

**错误**：我的代码用 `1e-5` 作为除数，导致解码后坐标超出范围
**正确**：除数应为 `1e-6`（即 `/ 1000000`）

### 解决方案
```javascript
// 腾讯地图官方文档：coors[i] = coors[i-2] + coors[i]/1000000
for (let i = 2; i < route.polyline.length; i++) {
    if (i % 2 === 0) {
        lat += route.polyline[i] / 1e6
    } else {
        lng += route.polyline[i] / 1e6
        routeCoords.push({ latitude: lat, longitude: lng })
    }
}
```

## 腾讯地图路线规划 duration 单位修复（17:43）

### 问题现象
18.6 公里路程显示预计 1 分钟，明显错误

### 根本原因
腾讯地图 WebService API 返回的 `duration` 单位已经是**分钟**，但代码错误地再除以 60：
```javascript
duration: route.duration ? Math.round(route.duration / 60) : 0
// 腾讯返回 49（分钟），49/60 ≈ 1 分钟 ❌
```

### 解决方案
移除错误的除法运算：
```javascript
duration: route.duration || 0  // 直接使用分钟单位
```
