## 修复 VM95:11 TypeError: Object(...) is not a function

### 问题分析
- 错误来自 `pages/index/index` 页面
- 根因1：`import` 使用相对路径 `../../utils/cloudCall.js`，uni-app 项目应使用 `@` alias
- 根因2：模板中 `getLevelClass(item.level)` 和 `getLevelText(item.level)` 直接调用方法（微信模板限制）
- 根因3：`computed` 中有未使用的 `formattedLevel` dead code

### 修复内容（index.vue）
1. ✅ import 路径从 `'../../utils/cloudCall.js'` 改为 `'@/utils/cloudCall.js'`
2. ✅ 删除未使用的 `formattedLevel` computed dead code
3. ✅ `getLevelClass` 和 `getLevelText` 从 `methods` 移到 `computed`（返回函数模式）
4. ✅ 模板中调用方式不变，computed 返回函数后模板调用 `getLevelClass(item.level)` 仍可正常工作

### 未解决
- `wx://not-found` 错误：经排查 uni-icons 组件已正确安装在 node_modules
- `componentNormalizer` 引用 HBuilderX 安装目录路径，需重新编译后验证
- `Object(...) is not a function` 在编译后 module 47（模板）中也可能有调用点，需重新编译项目验证

### 后续操作
